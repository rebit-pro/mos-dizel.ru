## Статус среды

Пользователь просил вести проектную документацию в WSL-проекте `dtcnab`.

Из текущей Codex-сессии WSL недоступен:

- `wsl.exe` сообщает, что установленные дистрибутивы не найдены;
- `\\wsl.localhost` и `\\wsl$` не существуют даже после запроса доступа вне sandbox.

Временный staging с той же структурой проекта создан здесь:

`C:\Users\User\Documents\ReBit Studio OS\projects\dtcnab`

Когда реальный WSL-путь будет доступен, папку `projects\dtcnab` нужно перенести в WSL-проект `dtcnab`, сохранив структуру.

## Границы текущей задачи

Входит:

- разобрать текущий сайт как источник контента;
- сохранить тексты, контакты, реквизиты, структуру, формы, изображения;
- подготовить варианты визуального направления для нового сайта на `mos-dizel.ru`;
- зафиксировать риски и открытые вопросы перед макетом.

Не входит в этот проход:

- разработка нового сайта;
- перенос DNS/домена;
- SEO-миграция со старого домена `top-diz.ru`;
- настройка аналитики и целей;
- юридическая проверка текстов/реквизитов;
- актуализация цен, если клиент их не подтвердил.

## Рекомендуемый следующий шаг

1. Клиент подтверждает актуальные цены, реквизиты, домен и список топлив.
2. Выбираем одно дизайн-направление из `.docs/design/design-directions.md`.
3. Готовим первый экран + ключевые секции в макете.
4. После утверждения макета переходим к frontend/SEO-структуре и миграционному чек-листу.
